# Security policy

## Supported versions

Only the latest tagged `0.1.x` release receives security fixes.

## Reporting

Report suspected vulnerabilities privately through GitHub Security Advisories for
`kotoba-lang/kagi`. Do not include real vault exports, passphrases, VMKs, private keys,
recovery material, or plaintext secrets. Use synthetic reproductions.

## Release status

`v0.5.0` is a technical preview, not an independently audited password-manager release.
Production adoption requires an external cryptography/application-security review and an
organization-specific recovery drill.
